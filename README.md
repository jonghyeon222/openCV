## 첫 번째 실습과제: 그림판 만들기
OpenCV의 마우스 이벤트와 키보드 입력을 활용하여 간단한 디지털 캔버스를 구현한 실습입니다.

1. 주요 기능
자유 드로잉: 마우스 왼쪽 버튼을 누른 상태로 움직이면 검은색 선이 그려집니다.
캔버스 초기화: 키보드 c 키를 누르면 화면이 다시 하얗게 지워집니다.
프로그램 종료: 키보드 q 또는 ESC 키를 누르면 창이 닫히고 프로그램이 종료됩니다.

2. 기술 스택 및 환경
Language: Python 3.9
Library:
 - OpenCV (4.13.0): 창 관리 및 마우스/키보드 이벤트 처리
 - Numpy (2.0.2): 512x512 크기의 하얀색 캔버스(행렬) 생성

3. 핵심 코드 설명
마우스 이벤트 처리:
 - cv2.EVENT_MOUSEMOVE와 cv2.EVENT_FLAG_LBUTTON의 비트 연산(&)을 통해 드래그 상태를 확인합니다.
키보드 이벤트 처리:
 - cv2.waitKey(0)를 통해 입력을 대기하며, ord('q')와 27(ESC)을 비교하여 종료 로직을 수행합니다.
 - if (key == ord('q') or key == 27): 구문을 사용하여 다중 종료 키를 지원합니다.

4. 실행 방법 
가상환경(Python 3.9) 활성화
pip install opencv-python numpy 설치
파이썬 파일 실행 후 마우스로 자유롭게 그림 그리기


## 두 번째 실습과제: 비트 연산을 활용한 이미지 합성

OpenCV의 HSV 색상 공간 변환과 비트 연산(AND, OR, NOT)을 활용하여 특정 배경을 제거하고 두 이미지를 합성하는 크로마키 기법을 구현한 실습입니다.

1. 주요 기능
색상 기반 마스킹: 특정 색상 범위(초록색)를 지정하여 배경을 분리하는 마스크를 생성합니다.
객체 추출 및 배경 제거: 생성된 마스크를 반전시켜 캐릭터 본체만 추출하고, 배경 이미지에서 해당 영역을 비워둡니다.
최종 이미지 합성: 추출된 캐릭터 본체와 영역이 확보된 배경 이미지를 결합하여 하나의 결과물을 만듭니다.

2. 기술 스택 및 환경
Language: Python 3.9
Library:
 - OpenCV (4.13.0): 이미지 프로세싱 및 비트 연산 처리
 - Numpy (2.0.2): 색상 범위 설정을 위한 배열(Array) 생성 및 행렬 연산

3. 핵심 코드 설명
색상 공간 변환 및 마스크 생성:
 - cv2.cvtColor(sonic, cv2.COLOR_BGR2HSV)를 통해 조명 변화에 강한 색상 검출 환경을 만듭니다.
 - cv2.inRange() 함수로 초록색 범위를 지정하여 배경은 흰색(255), 객체는 검은색(0)인 이진 마스크를 생성합니다.
비트 연산을 통한 합성 로직:
 - cv2.bitwise_not(mask1)을 통해 캐릭터 영역을 선택하는 반전 마스크를 생성합니다.
 - cv2.bitwise_and()로 배경이 제거된 캐릭터와 캐릭터 자리가 비워진 배경 이미지를 각각 준비합니다.
 - cv2.bitwise_or()를 사용하여 두 이미지를 병합함으로써 최종 합성 이미지를 도출합니다.

4. 실행 방법

## 세 번째 실습과제: 원근 변환 및 이미지 선명화 실습

OpenCV의 마우스 이벤트 처리와 기하학적 변환(Perspective Transform),
그리고 커널 기반의 샤프닝(Sharpening)필터를 활용하여 비스듬한 이미지를 정면 평면으로 바로잡고 화질을 개선하는 실습입니다.

1. 주요 기능 
인터랙티브 좌표 수집: 마우스 왼쪽 버튼 클릭을 통해 변환할 영역의 네 꼭짓점 좌표를 실시간으로 지정합니다.
원근 변환(Perspective Correction): 원근감이 포함된 사각형 영역을 사용자가 정의한 평면 좌표계로 재투영(Warping)하여 'Top-view' 이미지를 생성합니다.
이미지 선명화(Sharpening): 변환 및 확대 과정에서 발생하는 픽셀 뭉개짐 현상을 $3 \times 3$ 샤프닝 커널 연산을 통해 보정하여 시인성을 높입니다.

2. 기술 스택 및 환경
Language: Python 3.9
Library:OpenCV (4.13.0):
 - getPerspectiveTransform 및 warpPerspective를 이용한 기하학적 변환,
   filter2D를 이용한 커널 연산
 - Numpy (2.0.2): 좌표 행렬 생성 및 사용자 정의 샤프닝 커널(Matrix) 구성

3. 핵심 코드 설명
원근 변환 행렬 도출 및 적용:
 - cv2.getPerspectiveTransform(pts1, pts2)를 사용하여 사용자가 클릭한 4개의 점과 목표 평면 좌표 간의 3x3 변환 행렬을 계산합니다.
 - cv2.warpPerspective() 실행 시 flags=cv2.INTER_LANCZOS4를 적용하여 고품질의 픽셀 보간을 수행합니다.
커널 기반 후처리 보정:
 - np.array()를 사용하여 중앙값을 강조하고 주변을 감쇄시키는 샤프닝 마스크를 정의합니다.
 - cv2.filter2D() 함수로 컨볼루션(Convolution) 연산을 수행하여 결과 이미지의 경계선을 더욱 뚜렷하게 만듭니다.
비파괴 이미지 관리:
 - img.copy()를 사용하여 시각화용 이미지와 연산용 원본 이미지를 분리함으로써, 결과물에 마우스 클릭 표식이 남지 않도록 구현합니다.

4. 실행 방법
가상환경(Python 3.9) 활성화
상대 경로(../football-field.jpg)에 이미지 파일 존재 확인
파이썬 파일 실행 후 원본 이미지에서 좌상 -> 우상 -> 우하 -> 좌하 순서로 4개 지점 클릭
생성된 dst 창에서 원근 변환 및 선명화 처리가 완료된 최종 결과 확인


## 네 번째 실습과제: 환경 변화에 강인한 철길(Rail) 직선 검출

가우시안 블러(Gaussian Blur)와 확률적 허프 선 변환(Probabilistic Hough Line Transform)을 결합하여, 복잡한 노면 환경에서도 철길의 레일을 정확하게 추적하는 실습입니다.

1. 주요 기능
노이즈 억제 전처리: 가우시안 블러를 통해 자갈, 선로의 질감 등 자잘한 고주파 노이즈를 제거하여 깨끗한 에지를 확보합니다.
직선 성분 추출: 확률적 허프 변환을 사용하여 이미지 내에서 특정 길이 이상의 유의미한 직선(레일)만 선택적으로 검출합니다.
강건한 파라미터 설계: 고정된 높은 임계값에 의존하지 않고, 전처리와 기하학적 필터를 조합하여 다양한 조도 환경에서도 동작하도록 구현합니다.

2. 기술 스택 및 환경
Language: Python 3.9
Library:
 - OpenCV (4.13.0): Canny 에지 검출 및 HoughLinesP를 이용한 선 추출
 - Numpy (2.0.2): 행렬 복사 및 수학적 상수(np.pi) 지원

3. 핵심 코드 설명
다단계 노이즈 제어:
 - cv2.GaussianBlur(gray, (7, 7), 0)를 적용하여 에지 검출 전 단계에서 불필요한 질감을 부드럽게 처리합니다.
 - cv2.Canny(blur, 50, 150)를 통해 상한 임계값을 현실적인 범위로 설정하여 검출의 유연성을 높입니다.
확률적 허프 선 변환 최적화:
 - minLineLength=250을 설정하여 하늘의 전선이나 짧은 노이즈를 배제하고 긴 레일 선만 검출합니다.
 - maxLineGap=40을 통해 그림자나 이물질에 의해 끊긴 에지들을 하나의 매끄러운 직선으로 병합합니다.
안정적인 렌더링:
 - if lines is not None: 구문을 추가하여 선이 검출되지 않는 상황에서도 프로그램이 강제 종료되지 않도록 예외 처리를 수행합니다.

4. 실행 방법
가상환경(Python 3.9) 활성화 및 상대 경로(../rail.jpg) 이미지 확인
파이썬 파일 실행 후 edges 창의 에지 추출 상태와 img_copy 창의 최종 레일 검출 결과 비교 분석



## 다섯 번째 실습과제: 적응형 이진화와 모폴로지를 활용한 동전 계수

OpenCV의 적응형 이진화(Adaptive Threshold)와 모폴로지(Morphology) 연산을 활용하여, 조명 불균형이 있는 환경에서도 동전의 개수를 정확하게 자동으로 측정하는 실습입니다.

1. 주요 기능
조명 대응 이진화: 이미지의 국소 영역별로 임계값을 계산하여, 그림자가 지거나 빛 반사가 심한 환경에서도 동전의 형태를 보존합니다.
노이즈 제거 및 홀 채우기: 모폴로지 닫기(Closing) 연산을 통해 동전 표면의 무늬나 질감으로 인해 생기는 내부 구멍을 메워 단일한 덩어리(Blob)로 만듭니다.
객체 카운팅 및 결과 렌더링: 검출된 윤곽선(Contour)의 개수를 파악하여 이미지 상단에 분석 결과를 실시간으로 텍스트 출력합니다.

2. 기술 스택 및 환경
Language: Python 3.9
Library:
 - OpenCV (4.13.0): 적응형 이진화, 윤곽선 추출, 텍스트 렌더링 처리
 - Numpy (2.0.2): 모폴로지 연산을 위한 커널(Kernel) 생성 및 배열 연산

3. 핵심 코드 설명
적응형 이진화 및 반전 처리:
 - cv2.adaptiveThreshold()를 사용하여 blockSize=57, C=1 설정을 통해 국부적인 대비를 강조합니다.
 - cv2.THRESH_BINARY_INV를 적용하여 배경은 검은색(0), 객체(동전)는 흰색(255)으로 반전시켜 윤곽선 추출에 최적화된 상태를 만듭니다. 
모폴로지 연산을 통한 객체 정문화:
 - cv2.morphologyEx(..., cv2.MORPH_CLOSE, kernel)를 사용하여 이진화 과정에서 단절된 선을 연결하고 객체 내부를 꽉 찬 형태로 보정합니다.
윤곽선 데이터 구조 활용:
 - cv2.findContours()로 추출된 contours 리스트의 길이(len(contours))를 이용해 별도의 복잡한 알고리즘 없이 객체의 개수를 즉시 산출합니다.
 - cv2.putText()를 통해 정수형 데이터를 문자열로 변환하여 이미지 위에 검출 정보를 시각화합니다.

4. 실행 방법
가상환경(Python 3.9) 활성화
상대 경로(../coins.jpg)에 이미지 파일 존재 확인
파이썬 파일 실행 후 thresh(이진화), morph(보정), result(최종 카운팅) 창을 비교하며 검출 과정 확인



## 여섯 번째 실습과제: Haar Cascade와 비트 연산을 활용한 페이스 마스크(Tiger) 합성

OpenCV의 사전 학습된 Haar Cascade 분류기를 사용하여 실시간으로 얼굴을 탐지하고, 투명도(Alpha) 정보가 포함된 PNG 마스크 이미지를 얼굴 영역에 자연스럽게 합성하는 실습입니다.

1. 주요 기능
실시간 얼굴 검출: 웹캠 입력으로부터 사람의 얼굴 위치(x, y, w, h)를 실시간으로 추적합니다.
투명 마스크 합성: 마스크 이미지의 알파 채널을 분석하여, 배경은 투명하게 유지하고 호랑이 마스크 부분만 얼굴 위에 덧씌웁니다.
동적 크기 조절 및 위치 보정: 검출된 얼굴 크기에 맞춰 마스크 크기를 2배 확장(w2, h2)하고, 머리 윗부분까지 덮을 수 있도록 좌표를 오프셋(Offset) 조정합니다.

2. 기술 스택 및 환경
Language: Python 3.9
Library:
 - OpenCV (4.13.0): CascadeClassifier를 이용한 객체 탐지, 이미지 변환 및 비트 연산(bitwise_and, add) 처리
 - Numpy (2.0.2): 마스크 배열 슬라이싱 및 ROI(Region of Interest) 영역 연산

3. 핵심 코드 설명
알파 채널 분리와 이진 마스크 생성:
 - cv2.split(tiger)를 통해 B, G, R, A(알파) 채널을 분리합니다.
알파 채널을 임계값 처리(threshold)하여 마스크가 그려질 영역(흰색)과 투명하게 처리될 영역(검은색)을 구분하는 이진 마스크를 생성합니다.
Haar Cascade 분류기 활용:
 - cv2.CascadeClassifier에 미리 정의된 haarcascade_frontalface_default.xml을 로드하여 얼굴 영역을 사각형 형태로 검출합니다.
비트 연산을 이용한 정밀 합성:
 - cv2.bitwise_and(..., mask=mask_inv_re)를 사용하여 배경 이미지(웹캠)에서 마스크가 들어갈 자리를 비워둡니다.
 - cv2.bitwise_and(..., mask=mask_re)를 사용하여 마스크 이미지에서 실제 호랑이 그림 부분만 추출합니다.
 - cv2.add()를 통해 두 데이터를 합쳐 픽셀 깨짐 없는 깔끔한 합성 결과물(dst)을 생성합니다.

4. 실행 방법
가상환경(Python 3.9) 활성화
상대 경로(../tigermask.png)에 투명 배경을 가진 마스크 파일 존재 확인
파이썬 파일 실행 후 카메라를 응시하면 얼굴 위치를 따라다니는 호랑이 마스크 확인
종료 방법: 카메라 화면을 클릭한 후 키보드의 ESC 키를 누르면 프로그램이 종료됩니다.


## 일곱 번째 실습과제: HSV 색상 추적을 활용한 에어 캔버스(Air Canvas)

OpenCV의 HSV 색상 공간 변환과 모멘트(Moments) 계산을 활용하여 특정 색상의 물체를 추적하고, 실시간 웹캠 영상 위에 투명도가 적용된 선을 그려내는 디지털 캔버스 실습입니다.

1. 주요 기능
실시간 색상 추적: 초록, 파랑, 빨강 중 선택된 색상의 물체를 실시간으로 탐지하여 그 중심 좌표를 계산합니다.
동적 드로잉 및 모드 변경: 물체의 움직임을 따라 선이 그려지며, 키보드 입력(p)을 통해 인식 색상과 펜의 색상을 동시에 변경합니다.
투명도 합성(Alpha Blending): 그려진 선이 원본 영상을 가리지 않도록 80%의 불투명도를 적용하여 자연스러운 합성 효과를 구현합니다.
캔버스 제어: c 키를 눌러 저장된 좌표 리스트를 비움으로써 화면을 초기화할 수 있습니다.

2. 기술 스택 및 환경
Language: Python 3.9
Library:
 - OpenCV (4.13.0): HSV 변환, 이진화(inRange), 윤곽선 추출 및 무게중심 계산, 투명도 합성(addWeighted)
 - Numpy (2.0.2): 선을 그리기 위한 검은색 레이어(Canvas) 생성 및 행렬 연산

3. 핵심 코드 설명
객체 중심점(Centroid) 추출:
 - cv2.findContours로 추출된 가장 큰 윤곽선에 대해 cv2.moments(c)를 실행합니다.
 - 수식 cx = M10/M00, cy = M01/M00을 이용하여 객체의 수학적 무게중심 좌표를 도출합니다.
HSV 색상 최적화 및 피부색 회피:
 - OpenCV의 Hue 범위(0~179)를 고려하여 각 색상별 최적 임계값을 설정합니다.
 - 특히 빨간색 모드에서는 피부색(H: 0~20)과의 간섭을 피하기 위해 높은 Hue 대역(140~180)을 선택하여 정확도를 높였습니다.
레이어 기반 투명 선 구현:
 - 원본 프레임에 직접 그리지 않고, np.zeros_like로 생성한 별도의 검은색 레이어(canvas)에 cv2.line을 실행합니다.
 - cv2.addWeighted(frame, 1, canvas, 0.8, 0)를 통해 배경이 비쳐 보이는 투명한 선을 완성합니다.
상태 관리 및 초기화:
 - points 리스트에 좌표를 누적하여 프레임 간 연결성을 유지하며, 모드 변경 시 리스트를 초기화하여 이전 기록이 남지 않도록 관리합니다.

 - ESC: 프로그램 종료
